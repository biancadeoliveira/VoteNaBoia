Endpoint Alunos

Cadastra um novo aluno ao sistema.

O REQUEST deve enviar em formato JSON as seguintes informações:

         
    "NMAluno": "",
    "Email": "",
    "Senha": "",
    "SNEnviaEmail": "N",  
    "SNAtivo": "S"

Os campos SNEnviaEmail e SNAtivo aceitam 'S' ou 'N'

O RESPONSE será composto sempre por:

         
    "object": 
    "message": "",
    "skip": 0,
    "take": 0

Busca todos os alunos cadastrados no sistema, independente da turma a qual pertence

No REQUEST não é necessário enviar nenhuma informação, só acessar o endpoint

O RESPONSE será composto sempre por:

         
      "object": [{}, {}, {} ... {}]
      "message": "",
      "skip": 0,
      "take": 0

O Object será um conjunto de JSONs, exemplo:

        
    [
        {
            "IDAluno": 1
            "NMAluno": "Bianca",
            "Email": "bianca@bianca.com.br",
            "Senha": "12345",
            "SNEnviaEmail": "S",  
            "SNAtivo": "S" 
        }
    ]

Busca um aluno especifico passando o ID do aluno na rota

No REQUEST deve enviar em formato JSON as seguintes informações:

        
    {
        "IDAluno": 1
    }

O RESPONSE será composto sempre por:

         
    "object": {}
    "message": "",
    "skip": 0,
    "take": 0

O Object será um JSON, exemplo:

        
    {
        "IDAluno": 1
        "NMAluno": "Bianca",
        "Email": "bianca@bianca.com.br",
        "Senha": "12345",
        "SNEnviaEmail": "S",  
        "SNAtivo": "S" 
    }

Atualiza as informações de cadastro de um aluno especifico, passando o ID do aluno na rota

No REQUEST deve enviar em formato JSON as seguintes informações:

        
    {
        "IDAluno": 1
        "NMAluno": "Bianca",
        "Email": "bianca@bianca.com.br",
        "Senha": "12345",
        "SNEnviaEmail": "S",  
        "SNAtivo": "S" 
    }

É importante lembrar que o ID passado na rota e o ID passado por Json devem ser iguais, pois ambos referenciam o mesmo registro de aluno.
O Campo IDAluno não pode ser alterado.

O RESPONSE será composto sempre por:

         
    "object": null,
    "message": "",
    "skip": 0,
    "take": 0

Editar o campo SNAtivo do aluno, ou seja, o status, S = Ativo, N = Inativo

No REQUEST deve enviar em formato JSON as seguintes informações:

        
    {
        "IDAluno": 2,
        "SNAtivo": "N"
    }

O Campo IDAluno não pode ser alterado. O Campo SNAtivo só aceita 'S' ou 'N'.

O RESPONSE será composto sempre por:

         
    "object": null,
    "message": "",
    "skip": 0,
    "take": 0

Endpoint Restaurantes

Cadastra um novo restaurante no sistema. Os restaurantes sempre devem estar relacionacionados a uma turma, de forma que para um mesmo retaurante existir em duas turmas, ele deve ser cadastrado para as duas.

O REQUEST deve enviar em formato JSON as seguintes informações:

         
    "idTurma": 1,
    "nmNome": "Subway",
    "nmTipo": "Fastfood",
    "endereco": "Av Principal,123 - Centro",
    "noTelefone": "1412345678",
    "link": null,
    "email": "subway.com",
    "snAtivo": "S",
    "pagamentoRestaurante": [
        {
            "idFormaPagamento": 1
        },
        {
            "idFormaPagamento": 2
        },
        {
            "idFormaPagamento": 4
        }
    ]

É possivel cadastrar as formas de pagamento do restaurante por essa rota, ou por uma rota especifica de formas de pagamento.

O RESPONSE será composto sempre por:

         
    "object": null,
    "message": "",
    "skip": 0,
    "take": 0

Busca todos os restaurantes da turma, devemos passar na rota o id da turma

No REQUEST não é necessário enviar nenhuma informação, só acessar o endpoint

O RESPONSE será composto sempre por:

         
    "object": [{}, {}, {} ... {}]
    "message": "",
    "skip": 0,
    "take": 0
