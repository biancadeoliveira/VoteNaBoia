Endpoint Alunos

Cadastra um novo aluno ao sistema.

No REQUEST deve ser enviado, em formato JSON, as seguintes informações:

         
    "NMAluno": "",
    "Email": "",
    "Senha": "",
    "SNEnviaEmail": "N",  
    "SNAtivo": "S"

Os campos SNEnviaEmail e SNAtivo aceitam 'S' ou 'N'

O RESPONSE será composto sempre por:

         
    "object": 
    "message": "",
    "skip": 0,
    "take": 0

Busca todos os alunos cadastrados no sistema, independente da turma a qual pertence

No REQUEST não é necessário enviar nenhuma informação, só acessar o endpoint

O RESPONSE será composto sempre por:

         
      "object": [{}, {}, {} ... {}]
      "message": "",
      "skip": 0,
      "take": 0

O Object será um conjunto de JSONs, exemplo:

        
    [
        {
            "IDAluno": 1
            "NMAluno": "Bianca",
            "Email": "bianca@bianca.com.br",
            "Senha": "12345",
            "SNEnviaEmail": "S",  
            "SNAtivo": "S" 
        }
    ]

Busca um aluno especifico passando o ID do aluno na rota

No REQUEST deve enviar em formato JSON as seguintes informações:

        
    {
        "IDAluno": 1
    }

O RESPONSE será composto sempre por:

         
    "object": {}
    "message": "",
    "skip": 0,
    "take": 0

O Object será um JSON, exemplo:

        
    {
        "IDAluno": 1
        "NMAluno": "Bianca",
        "Email": "bianca@bianca.com.br",
        "Senha": "12345",
        "SNEnviaEmail": "S",  
        "SNAtivo": "S" 
    }

Atualiza as informações de cadastro de um aluno especifico, passando o ID do aluno na rota

No REQUEST deve ser enviado, em formato JSON, as seguintes informações:

        
    {
        "IDAluno": 1
        "NMAluno": "Bianca",
        "Email": "bianca@bianca.com.br",
        "Senha": "12345",
        "SNEnviaEmail": "S",  
        "SNAtivo": "S" 
    }

É importante lembrar que o ID passado na rota e o ID passado por Json devem ser iguais, pois ambos referenciam o mesmo registro de aluno.
O Campo IDAluno não pode ser alterado.

O RESPONSE será composto sempre por:

         
    "object": null,
    "message": "",
    "skip": 0,
    "take": 0

Editar o campo SNAtivo do aluno, ou seja, o status, S = Ativo, N = Inativo

No REQUEST deve ser enviado, em formato JSON, as seguintes informações:

        
    {
        "IDAluno": 2,
        "SNAtivo": "N"
    }

O Campo IDAluno não pode ser alterado. O Campo SNAtivo só aceita 'S' ou 'N'.

O RESPONSE será composto sempre por:

         
    "object": null,
    "message": "",
    "skip": 0,
    "take": 0

O RESPONSE será composto sempre por:

Endpoint Voto Semanal

Insere um novo voto para o aluno no periodo da turma.

No REQUEST não é necessário enviar nenhuma informação, só acessar o endpoint.

Validações:
1 - O Aluno pode votar em n restaurantes, sendo n a quantidade de dias que terá votação na semana (de segunda a sexta), conforme configurado nas configurações da turma.
2 - Só aceitam novos votos periodos que estão abertos (SNAtivo = 'S') e não processados (SNProcessado = 'N')
3 - O Aluno não pode votar mais de uma vez no mesmo restaurante no mesmo periodo semanal

O RESPONSE será composto sempre por:

         
      "object": null,
      "message": "",
      "skip": 0,
      "take": 0

Busca todos os votos do aluno no periodo da turma indiicado

No REQUEST não é necessário enviar nenhuma informação, só acessar o endpoint e passar no {idPeriodo} o periodo que deseja buscar as votações e o no {idTurmaAluno} o id que representa o vinculo entre aluno e turma.

O RESPONSE será composto sempre por:

         
      "object": [{}, {}, {} ... {}]
      "message": "",
      "skip": 0,
      "take": 0

O Object será um conjunto de JSONs, exemplo:

        
    [
        {
            "idVotoSemanal": 7,
            "idRestaurante": 2,
            "idPeriodo": 4,
            "idTurmaAluno": 5,
            "dhInclusao": "2020-06-28T23:29:06.573"
        }
    ]

Processa os votos semanais do periodo passado pelo ID.

No REQUEST não é necessário enviar nenhuma informação, só acessar o endpoint e passar no {IDPeriodo} o id da periodo a ser processado.

Validações:
1 - O Período necessita ter o campo SNProcessado = 'N' para contabilizar os votos da semana.
2 - O Período só será computado uma vez, e terá o SNProcessado alterado para 'S'.

O RESPONSE será composto sempre por:

         
    "object": null,
    "message": "",
    "skip": 0,
    "take": 0
